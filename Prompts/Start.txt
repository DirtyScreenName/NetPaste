Build the first releasable version of a small product with the working name “NetPaste.”

MISSION

NetPaste helps network engineers clean messy CLI output before placing it into tickets, documentation, GitHub issues, Slack, or AI tools.

The immediate objective is to ship a small, polished, working static web app. Do not build the Chrome extension yet. Structure the core processing logic so it can be reused by a Manifest V3 Chrome extension later.

This is a completion project, not a startup platform. Keep the scope aggressively small.

PRODUCT BEHAVIOR

The user should be able to:

1. Paste network CLI output into an input field.
2. Click “Clean Output.”
3. See cleaned output in a separate output field.
4. See warnings for potentially sensitive information found in the text.
5. Copy the cleaned output as plain text.
6. Copy the cleaned output as a Markdown code block.
7. Clear both fields.
8. Load a safe built-in example for demonstration.

CLEANING RULES

Implement deterministic, conservative text-cleaning functions that:

- Normalize CRLF and CR line endings to LF.
- Remove ANSI escape and terminal color/control sequences.
- Remove null characters and obvious non-printing terminal artifacts.
- Remove common pagination markers such as:
  - --More--
  - ---- More ----
  - <--- More --->
- Handle backspace characters commonly produced by terminal pagination.
- Remove trailing whitespace from every line.
- Collapse runs of more than two blank lines down to two.
- Remove leading and trailing blank lines.
- Preserve indentation and meaningful spacing inside configuration output.
- Avoid destructive transformations when uncertain.

Do not modify IP addresses, hostnames, passwords, secrets, or other values automatically. Detection and redaction are separate concerns.

SENSITIVE-DATA DETECTION

Analyze the original and cleaned text locally and flag potential sensitive information without automatically altering it.

Detect and report, at minimum:

- IPv4 addresses
- IPv6 addresses where reasonably identifiable
- MAC addresses
- Email addresses
- URLs
- Hostname-like device prompts
- Lines containing words or configuration patterns associated with:
  - password
  - passwd
  - secret
  - enable secret
  - username
  - community
  - snmp-server community
  - private key
  - pre-shared key
  - preshared key
  - key-string
  - API key
  - token
  - bearer
  - authentication key

Each finding should include:

- Category
- Line number
- A short, safely escaped preview
- Severity of “Review” or “High review priority”

Do not display secret values more prominently than they already appear. Never log pasted content to the console.

Show this message near the results:

“NetPaste flags common sensitive patterns but cannot guarantee that all confidential information has been identified. Review the output before sharing it.”

PRIVACY AND SECURITY REQUIREMENTS

- All processing must occur inside the browser.
- No backend.
- No API calls.
- No analytics.
- No telemetry.
- No cookies.
- No localStorage or sessionStorage.
- No upload functionality.
- No external fonts, scripts, stylesheets, images, or CDNs.
- Do not send pasted content anywhere.
- Do not use innerHTML with user-controlled text.
- Do not add an AI model or OpenAI API.
- Include a plain-language privacy page that accurately explains the local-only behavior.
- Add an obvious “Processed locally in your browser” indicator.

MARKDOWN COPYING

When copying as Markdown:

- Wrap the cleaned output in a fenced code block.
- Use a safe fence length if the content itself contains backticks.
- Use “text” as the default language identifier.
- Preserve the cleaned output exactly inside the fence.

TESTING

Write meaningful unit tests covering:

1. ANSI sequence removal.
2. Pagination-marker removal.
3. Backspace artifact handling.
4. Line-ending normalization.
5. Whitespace preservation.
6. Blank-line cleanup.
7. IPv4 detection.
8. MAC-address detection.
9. Password/secret configuration detection.
10. SNMP community detection.
11. Avoiding obvious false positives where practical.
12. Markdown output containing embedded backticks.
13. Empty input.
14. Large multiline input.

Include at least one sanitized Cisco-style example and one generic terminal-output example in tests.
