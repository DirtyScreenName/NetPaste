AI prompts need technical context, not production identifiers

Network troubleshooting depends on relationships: the same address appears in an interface, a route, and a log message; the same hostname appears in prompts and neighbor output. Deleting those values entirely can make the evidence hard to reason about. Sending the original values may reveal topology, customers, credentials, or operational naming conventions.

NetPaste's Prepare for AI workflow runs locally. It applies the AI Prompt profile, selects recommended findings, preserves structure, and copies a fenced Markdown payload with a short safety header. It does not call an AI service or transmit the source text.

Use stable tokens to preserve relationships

Sanitized example input
Jul 17 10:14:02 CORE-01:
BGP-5-ADJCHANGE: neighbor 203.0.113.44 Down

CORE-01# show ip route 203.0.113.44
Routing entry for 203.0.113.44/32
  Known via "bgp 64512"
  Last update from 198.51.100.7
Stable-token output
Jul 17 10:14:02 <HOST-1>:
BGP-5-ADJCHANGE: neighbor <IP-1> Down

<HOST-1># show ip route <IP-1>
Routing entry for <IP-1>/32
  Known via "bgp <ASN-1>"
  Last update from <IP-2>

Stable tokens retain equivalence inside the current text without exposing the original matched value. The output remains editable, so you can add the troubleshooting question and remove irrelevant sections before copying.

Recommended workflow

  1. Paste only the evidence needed for the troubleshooting question.
  2. Choose the matching document mode and vendor rule pack.
  3. Select the AI Prompt profile or use Prepare for AI.
  4. Enable stable tokens when repeated values carry useful relationships.
  5. Review credentials, certificate material, identities, addresses, and metadata.
  6. Read the complete editable output before copying Markdown into the destination.

What NetPaste cannot decide for you

Pattern detection cannot understand every proprietary identifier or contractual restriction. A harmless-looking interface description may still identify a customer. A private address may still reveal an internal topology. The Safe Share Score and profile defaults help prioritize review, but the engineer remains responsible for the final disclosure decision.

Destination matters: review the AI provider's own retention, training, account, and enterprise-data terms before sending the prepared result. NetPaste only controls the local preparation step.

Local-first boundaries

NetPaste uses no AI integration, external network API, account, analytics, or intentional pasted-content storage. The extension does not read webpages. Copying places the reviewed Markdown on the system clipboard at your request.
