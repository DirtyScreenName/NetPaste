# NetPaste Distribution Playbook

## Positioning

NetPaste is the local-first redaction workbench for network configurations,
CLI output, troubleshooting logs, and technical tickets.

Lead with the user's outcome: share useful troubleshooting context without
casually exposing credentials or infrastructure details. Support that promise
with proof: local processing, no host permissions, no webpage reading, no
accounts, and no analytics.

## Search themes

Use these phrases naturally when they fit. Do not repeat them mechanically.

- network configuration redaction
- CLI output cleaner
- sanitize Cisco configuration
- redact IP addresses and credentials
- network log sanitizer
- local data redaction extension
- clean terminal output for tickets
- redact configuration before sharing

## Channel plan

### Chrome Web Store

- Keep the first sentence outcome-focused and readable in search results.
- Use all five screenshots to tell one workflow: paste, detect, select, review,
  and copy.
- Reply to substantive reviews and turn recurring problems into regression tests.
- Review the store's impressions, listing visits, installs, and uninstall trend
  weekly. Do not add application telemetry.

### Protocols & Packets

- Keep NetPaste in the parent site's Tools section with a direct store link and
  web-app link.
- Publish practical examples through existing parent-brand channels rather than
  adding a NetPaste blog.
- Demonstrate sanitized, realistic workflows for TAC cases, change tickets,
  GitHub issues, and AI prompts.

### Professional communities

- Share a useful redaction checklist first and NetPaste second.
- Target network engineering, security operations, sysadmin, and technical
  support communities where self-promotion is permitted.
- Use sanitized demonstrations and disclose the creator relationship.
- Avoid mass posting, purchased reviews, incentivized ratings, or fabricated
  adoption claims.

## Four-week launch cadence

1. Week 1: announce the public extension with a 30-second workflow demo.
2. Week 2: publish a before-and-after Cisco or firewall troubleshooting example.
3. Week 3: publish the local-only architecture and permission rationale.
4. Week 4: share improvements made from early user feedback.

Reuse each core story as a parent-site tool update, LinkedIn post, short video,
and community-specific post. Rewrite the opening for each audience rather than
posting identical copy everywhere.

## Measurement without product telemetry

Record a weekly snapshot of Chrome Web Store impressions, listing visitors,
installs, uninstalls, rating, and review themes. Track website search performance
through the hosting platform and Google Search Console only if that external
measurement is approved separately; do not add tracking scripts to NetPaste.

The primary early signals are listing-view-to-install conversion, uninstall
trend, repeat review themes, and the number of users who report a successfully
sanitized real workflow.

## Release checklist

- Confirm website and store copy describe the current product accurately.
- Validate structured data and social-card previews.
- Submit `sitemap.xml` in Google Search Console.
- Confirm the Chrome Web Store link resolves in all public calls to action.
- Test keyboard operation, narrow layouts, and clipboard failure handling.
- Confirm no new application requests, persistence, permissions, or telemetry.
